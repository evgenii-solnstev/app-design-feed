# Design Feed — контекст проекта

**Дата снимка контекста:** 26 февраля 2025

---

## 1. Назначение проекта

**Design Feed** — внутренний инструмент для дизайн-команд. Веб-приложение, в котором дизайнеры публикуют и просматривают скриншоты фреймов из Figma. Публикация возможна через Figma-плагин; просмотр — в браузере (сетка/лента, полноэкранный просмотр с миниатюрами и pull-to-switch).

## 3. Архитектура данных

### База (Prisma)

- **Author** — автор фрейма: `id`, `name` (unique), `createdAt`.
- **Frame** — один фрейм: `id`, `mediaUrl`, `mediaType`, `aspectRatio`, `width`, `height`, `figmaUrl`, `authorId` → Author, `tags` (String[]), `comment`, `createdAt`.

### API

- **GET /api/frames** — список фреймов с cursor-based пагинацией. Параметр `cursor`. Ответ: `{ items: Frame[], nextCursor: string | null }`.
- **POST /api/frames** — создание фрейма (multipart): `file`, `authorName`, `figmaUrl`. Загрузка файла в Vercel Blob, запись в БД. CORS и OPTIONS настроены для запросов из Figma-плагина.

### Типы (lib/types.ts)

- `Author`, `Frame`, `FramesResponse` — соответствуют Prisma и ответам API.

## 4. Структура приложения

### Страницы и layout

- **app/page.tsx** — единственная страница (client): заголовок "Design Feed", переключатель Сетка/Лента справа, один общий скролл (header не фиксирован). Кнопка «вверх» при скролле > 1 viewport (слева внизу, 32px от края).
- **app/layout.tsx** — корневой layout.
- **app/globals.css** — темы, переменные, ключевые кадры для анимации стрелки в лайтбоксе.

### Компоненты

- **components/frame-feed.tsx** — лента: загрузка GET /api/frames, infinite scroll (sentinel + IntersectionObserver), режимы grid / strip. Рендер `FrameCard` (сетка) или `StripSlide` (лента), при клике по картинке открывается `FrameLightbox`.
- **components/frame-card.tsx** — карточка в сетке: изображение (fit по ширине контейнера, обрезка по высоте, object-cover), автор (аватар по инициалам), дата, ссылка на Figma, теги. Без обводки и тени.
- **components/frame-lightbox.tsx** — полноэкранный просмотр: одна картинка fit по ширине, скролл по высоте; миниатюры слева; pull-to-switch на границах скролла (вверху — prev, внизу — next) со стрелкой и анимацией выхода.
- **StripSlide** (в frame-feed.tsx) — один блок в режиме «Лента»: изображение + имя, дата, ссылка на Figma. Без разделителей, уменьшенные отступы между блоками.

### Figma-плагин (figma-plugin/)

- **manifest.json** — конфигурация плагина.
- **code.js** — экспорт выбранного фрейма в JPG (макс. ширина 1200px), отправка на POST /api/frames, автор из `figma.currentUser.name`.
- **ui.html** — UI плагина: описание, кнопка «Поделиться в ленте», retry до 3 попыток. `allowedDomains` только с `https://`.

## 5. Принятые решения и последние апдейты

### Лайтбокс (full-screen)

- Один режим: картинка **fit по ширине**, высота по контенту, **скролл по вертикали**. Кнопок zoom in/out нет.
- **Миниатюры** слева: размер 74×55 px, отступ слева 24 px, между миниатюрами 16 px. Прозрачность: 0.3 обычные, 0.7 выбранная. Всегда видны.
- **Pull-to-switch**: срабатывает только на границах скролла (вверху — тяга вниз → предыдущий кадр; внизу — тяга вверх → следующий). В середине контента — обычный скролл. Порог переключения 60 px, стрелка 16 px, анимация выхода 300 ms. Используются refs для актуального currentIndex/onIndexChange, чтобы не было «залипания» после первого срабатывания.
- Нативный `touchmove` с `passive: false` и `preventDefault` только при тяге на границе, чтобы не конфликтовать со скроллом страницы.
- При смене кадра — короткая анимация появления (opacity 200 ms).

### Главная страница и лента

- **Фон сервиса:** #F5F5F5 (в :root и на странице).
- **Header:** без нижнего border, заголовок 40 px bold, отступ сверху 64 px, снизу 40 px; переключатель Сетка/Лента выровнен по вертикали по центру.
- **List-view (Лента):** без разделителей (border-b убран), отступы между блоками уменьшены на ~30% (py-6, gap-3, sm:gap-6).
- **Card-view (Сетка):** карточки без обводки (border-0), изображение — fit по ширине контейнера с обрезкой высоты (object-cover, object-left-top), без белых полей внутри карточки.

### Общее

- Один общий скролл: header и контент скроллятся вместе.
- Кнопка «Прокрутить вверх» — при скролле больше одного viewport, слева внизу (32 px от края), прозрачная, со стрелкой вверх.
- Escape закрывает лайтбокс; стрелки вверх/вниз переключают кадр с клавиатуры.
